# BuildCores API Product Information

BuildCores is a 3D platform for PC and setup building. The partner API offering
lets retailers, brands, and commerce platforms use BuildCores rendering,
catalog, compatibility, and integration services inside their own customer
experiences.

<figure className="doc-figure">

  <figcaption>Cloud-rendered BuildCores PC build output.</figcaption>
</figure>

## Integration Pillars

### Customer-Facing Technology

- Efficient 3D rendering technology for mobile and desktop experiences.
- Customizable 3D and 2D user interfaces for commerce, configurator, and product-detail-page workflows.
- Support for complete PC build rendering, setup rendering, and single product rendering, depending on the integration.

### Backend Technology

- Asset streaming and render pipelines for 3D models, sprites, videos, and static media.
- A large product database with product metadata, compatibility data, identifiers, and filtering fields.
- Globally cached media delivery for generated render assets.

### Reliability And Support

- Partner support during integration and production rollout.
- API key management, render usage monitoring, and issue escalation through partner support channels.
- Product and technical feedback can be routed into the BuildCores roadmap.

## Available Services

### Cloud-Rendered 3D Configurator

  <img src={medalConfigurator} alt="Medal.tv configurator powered by BuildCores" />
  <figcaption>Example of a cloud-rendered configurator powered by BuildCores on Medal.tv.</figcaption>

- Designed to look strong and stay performant across device classes.
- Cloud-rendered output works across mobile devices, web UIs, and native UI frameworks.
- Supports whole PC build rendering, setup rendering, and single product rendering.
- Supports complex RGB effects and case fan configuration when the selected parts and saved interactive state include those placements.

### Locally Rendered Components

<figure className="doc-figure">
  <img src={localConfigurator} alt="Locally rendered BuildCores configurator UI" />
  <figcaption>Example of a locally rendered configurator on BuildCores.com.</figcaption>
</figure>

- Uses the same browser rendering family that powers interactive 3D on BuildCores.com.
- Provides fully interactive 3D scenes and customization for qualified partner integrations.
- Supports whole PC build, setup, and single product rendering workflows when the integration is scoped for local rendering.
- Can support RGB effects, case fan configuration, and AR-style workflows where the target platform and partner package support them.

<p className="pricing-note">For pricing information, reach out to harsh@buildcores.com.</p>

Locally rendered integrations are separate from the cloud-rendered
`@buildcores/render-client` flow documented in the technical guide.

### Product Database

- Rich product specifications, attachments, filtering information, and identifiers.
- Compatibility data for PC build workflows, including wattage estimation.
- Price-comparison tooling can be made available for partner integrations.

<p className="pricing-note">For pricing information, reach out to harsh@buildcores.com.</p>

### 3D Modeling

- BuildCores can model brand products and add optimized assets to the partner configurator.
- Assets are prepared for efficient rendering and modular use in configurators.
- High-fidelity renders can be produced for product imagery.

<p className="pricing-note">For pricing information, reach out to harsh@buildcores.com.</p>

For retailers and brands that want BuildCores inside an existing commerce stack
without rebuilding the full experience in-house, BuildCores can provide custom
